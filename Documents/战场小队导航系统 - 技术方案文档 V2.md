# TireflySquadNavigation 插件 —— 技术方案文档 V2

> **版本说明**：本文档整合了原始技术方案文档与两份补充修正文档的全部内容，是当前唯一权威的实现参考。

>
> **目标表现**：实现类似《云顶之弈》《英雄联盟》中 AI 战斗单位的战场移动行为——自动导航至攻击目标附近，进入攻击距离后停止并攻击；移动过程中能平滑绕开其他移动中的单位和站桩战斗中的单位，彼此之间不推挤、不重叠、不卡死。

## 一、插件定位与核心约束

### 1.1 使用场景定位

本插件面向**小规模战队间的战斗导航**，以自走棋（Auto Chess）类游戏为典型场景。

三条核心约束，所有设计决策均由此推导：

```
约束一：规模约束
  单位数量级：十几到几十个
  → 优先保证正确性，不追求大规模极致性能优化

约束二：复用约束
  跨项目插件，不能假设宿主项目的具体实现
  → 插件内严禁 Cast 到任何宿主类
  → 通过 ITsnTacticalUnit 接口与宿主项目通信

约束三：地图约束
  主要面向棋盘格 / 规整开阔地
  → NavModifier 默认策略为不可通行（Impassable）
  → 保留高代价模式供复杂地形项目覆盖
```

### 1.2 插件提供的内容

| 类型 | 内容 |
|------|------|
| 接口（UInterface） | `ITsnTacticalUnit` — 宿主项目实现，插件通过接口读取单位属性 |
| 组件（Component） | 主要挂到宿主项目的 Character / Actor 子类上；其中 `TsnTacticalMovementComponent` 仅适用于 `ACharacter` |
| 世界子系统（WorldSubsystem） | 自动存在，无需手动创建 |
| 行为树节点（BT Task / Decorator） | 运行时模块直接提供并维护；当前演示资产与自动化测试仍以 BT / Blackboard 为基础 |
| 状态树节点（StateTree Task / Evaluator / Condition） | 运行时模块提供 5 个任务 + 2 个评估器 + 5 个条件；测试模块额外提供 2 个 demo-only Task 用于演示场景复现（详见 §七·补） |
| Editor-only 测试模块 | `TireflySquadNavigationTest` — 提供演示 Actor、自动化测试、测试专用 BT / StateTree 节点，不进入 Shipping |
| NavArea 类 | 项目中直接引用 |

## 三、行为分析与问题拆解

### 3.1 目标行为拆解

| 层次 | 行为描述 | 核心难点 |
|------|---------|---------|
| **宏观·寻路** | 选定目标 → 规划路径 → 沿路径移动 → 进入攻击距离后停止攻击 | 路径规划 |
| **中观·避障** | 移动路径上遇到其他单位时平滑绕开而非推挤/卡住 | 局部动态避障 |
| **微观·协调** | 多个单位同时移动时彼此不重叠、不卡死、不堵塞 | 群体运动协调 |

### 3.2 战场上单位的两种状态

| 类型 | 状态 | 速度 | 导航身份 |
|------|------|------|---------|
| **移动中的单位** | 正在向目标导航 | $v \neq 0$ | Crowd Agent（参与速度协商） |
| **站桩战斗的单位** | 已到达攻击位，正在攻击 | $v = 0$ | 临时动态障碍物 |

### 3.3 核心技术挑战

1. **RVO / DetourCrowd 对零速度单位失效**：算法基于"速度空间"协商，站桩单位速度为零时退化为一个点，无法有效作为障碍处理
2. **多个不同攻击距离的单位围攻同一目标**：近战、长矛、弓箭手攻击距离各不相同，需要按距离分环站位
3. **目标移动时槽位追踪点失效**：若把槽位当作固定世界点追逐，目标移动后旧快照会迅速过时，导致追击轨迹扭曲
4. **排斥力与路径跟随的 Velocity 写入冲突**：直接修改 `Velocity` 会被 `CalcVelocity()` 覆盖
5. **排斥力在 NavModifier 已生效范围内仍干扰路径**：外层排斥力过强会破坏路径跟随的主导权

### 4.2 双层防线设计

应对"移动单位需要绕开站桩单位"这一核心难题：

```
第一层防线：NavModifier（路径规划层）
  站桩单位脚下区域标记为 TsnNavArea_StanceUnit（默认不可通行）
  移动单位寻路时路径直接绕开这些区域

          B⊘    C⊘              ⊘ = 不可通行区域
  A ─ ─ ╮            ╭─ → Target
        ╰── ── ── ──╯          A 的路径自动绕行
                D⊘

第二层防线：双阶段排斥力（运行时兜底）
  内层（0 ~ NavModifierRadius）：NavMesh 延迟时的穿入 → 全力排斥
  外层（NavModifierRadius ~ RepulsionRadius）：路径规划已覆盖 → 极微弱引导（上限10%）
  排斥力通过 SetRepulsionVelocity 合法注入，不与路径跟随冲突
```

### 4.3 交战槽位设计（多环角度分配）

不同攻击距离的单位按自身 `GetEngagementRange()` 分配到对应的环：

```
改进前（单环固定格子）：
        ②     ①
      ③    [目标]   ⑧     ← 所有人在同一个环上，不区分攻击距离
        ④     ⑦
          ⑤ ⑥

改进后（多环角度分配）：
    弓 ─ ─ ─ ─ ─ ─ ─ ─ ─ 弓
    ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─     ← 外环：远程（半径 = AttackRange - 20）
       矛 ─ ─ ─ ─ ─ 矛
       ─ ─ ─ ─ ─ ─ ─ ─         ← 中环：长矛
          剑 ─ ─ 剑
          ─ [目标] ─             ← 内环：短剑
          剑 ─ ─ 剑
```

### 4.4 三阶段行为模型

**设计动机**：若等到单位进入攻击距离才分配槽位，所有单位会同时拥挤到目标附近再展开；若在距离非常远时就分配，目标一旦移动槽位坐标立刻严重漂移。因此引入中间的「预战斗距离」作为槽位分配的触发点。

```
[距离 > PreEngagementRadius]         → 阶段1：远距离追击
   ↓ 进入 PreEngagementRadius
[AttackRange < 距离 ≤ PreEngagementRadius] → 阶段2：预占位接近
   ↓ 进入 AttackRange
[距离 ≤ AttackRange]                 → 阶段3：就地站桩攻击
```

各阶段行为规则：

| 阶段 | 距离区间 | 行为 | 槽位操作 |
|------|---------|------|---------|
| **远距离追击** | dist > PreEngagementRadius | 自由追击目标，不做槽位操作 | 无 |
| **预占位接近** | AttackRange < dist ≤ PreEngagementRadius | 向 `TsnEngagementSlotComponent` 申请槽位，移动至槽位世界快照；目标每移动一定距离时基于已持有的本地槽位参数重算当前快照 | 申请并持有 |
| **就地站桩** | dist ≤ AttackRange（进入阶段2 时） | 跳过槽位申请，在当前位置直接进入站桩攻击 | 无 |

**退出阶段2 的三个条件**（满足任一均会离开本阶段，不允许无限等待阻塞攻击）：
- **(a) 到达槽位** → `Succeeded`：与槽位世界快照距离 ≤ 到达容差
- **(b) 中途进入攻击距离** → `Succeeded`：目标向自身靠近，不再需要继续移动到槽位
- **(c) 超时** → `Failed`：接近阶段累计时间超过上限，路径异常时的安全兜底；由 BT 失败分支处理恢复逻辑

**阶段3 的槽位释放**：就地站桩时没有占用任何槽位，之后执行释放操作为空操作，无副作用。

> `PreEngagementRadius = AttackRange × 系数`（系数可配置，默认 1.5）。  
> 系数越大，单位越早开始分散占位，战场布局越舒展；系数过大会使追击阶段过短，目标移动时频繁触发重算。

## 五、UE5.7 内置功能使用方式

### 5.1 RecastNavMesh —— 全局路径规划

- `UAIController::MoveToLocation / MoveToActor` 封装完整寻路 + 移动
- **配置要点**：场景放置 `NavMeshBoundsVolume`，配置 `Agent Radius`、`Agent Height`

### 5.2 DetourCrowd —— 群体调度 + 局部避障

- 在 AIController 中将 `PathFollowingComponent` 替换为 `UCrowdFollowingComponent`
- **关键状态**：
  - `ECrowdSimulationState::Enabled` —— 正常 Agent（移动中单位）
  - `ECrowdSimulationState::ObstacleOnly` —— 仅作为障碍物（站桩单位）
- **重要**：启用 DetourCrowd 后必须**关闭** CharacterMovementComponent 内置 RVO Avoidance。  
  两者同时开启会引发三个问题：
  - **数据污染**：`CalcAvoidanceVelocity` 在 `CalcVelocity` 末尾执行，此时 `Velocity` 已是 `ApplyRequestedMove` 写入的 DetourCrowd 速度 $V_{crowd}$。`AvoidanceManager->GetAvoidanceVelocityForComponent(this)` 采用 ORCA（速度障碍）数学，将当前 `Velocity`（即 $V_{crowd}$）作为"期望速度"，在其他所有 Agent 的速度障碍锥体中寻找距离 $V_{crowd}$ 最近的可行速度 $V_{rvo}$，然后 `Velocity = NewVelocity` 直接覆盖。这在数学上等价于把"已规避过的速度"当作"自然移动意图"再规避一次，输出完全无意义。
  - **双重注册**：`CalcAvoidanceVelocity` 末尾无条件调用 `AvoidanceManager->UpdateRVO(this)`，将本单位当前速度注册进 `AvoidanceManager` 的全局 RVO 状态。场景中所有单位自己的 `CalcAvoidanceVelocity` 都会读到本单位的 RVO 数据，产生额外规避反应——跟 DetourCrowd 已在全局 Agent 层做的规避完全叠加，导致过度绕行和速度抖动。
  - **Lock 机制干扰**：`CalcAvoidanceVelocity` 开头是 `if (AvoidanceLockTimer > 0) { Velocity = AvoidanceLockVelocity; }` 分支——若处于锁定期，**直接用锁定速度覆盖 `Velocity`，跳过所有 RVO 重新计算**。这一覆盖发生在 `ApplyRequestedMove` 之后（`CalcVelocity` 内的顺序：`ApplyRequestedMove` → 加速度/制动 → `CalcAvoidanceVelocity`），DetourCrowd 刚写入的速度被本帧完全丢弃。触发规避时锁定 `LockTimeAfterAvoid` 时长；即便 RVO 不改变方向，也会设置约 10 ms 的 `LockTimeAfterClean` 清洁锁——即**每帧速度非零时几乎必然产生短暂锁定**，DetourCrowd 输入周期性失效，单位表现为"冻结-突进"跳帧感。

### 5.3 NavModifierComponent —— 动态标记导航区域

- 运行时通过 `SetCanEverAffectNavigation()` 开关，**不做创建/销毁**（避免频繁 GC）
- `BeginPlay` 时预创建，初始状态关闭；进入 Stance 时开启，退出时延迟关闭
- 开关后均触发相关 NavMesh Tile 局部重建

### 5.4 CharacterMovementComponent —— 角色移动

- 重写 `CalcVelocity()`，在 `Super` 之后统一注入排斥力（UE 5.7 中 `RequestDirectMove` 只是输入端，最终都经 `CalcVelocity` 消费）
- AI 战斗单位的 CapsuleComponent Pawn 碰撞通道设为 `ECR_Overlap`（由导航/避障系统管理间距，而非物理碰撞）

> **`bUseAccelerationForPaths` 兼容性说明**：
> 插件**不强制设置** `bUseAccelerationForPaths`，对 `true`/`false` 两种模式均兼容。
>
> **两条路径的机制**：
> - `false`（默认）：CrowdFollowingComponent 调用 `RequestDirectMove(NewVelocity)` → 写入
>   `RequestedVelocity + bHasRequestedVelocity = true`，下一帧 `CalcVelocity` →
>   `ApplyRequestedMove` 消费。`ApplyRequestedMove` 内部还有两个子分支，由
>   `ShouldComputeAccelerationToReachRequestedVelocity()` 决定：
>   - **加速分支**（`bRequestedMoveUseAcceleration = true`（构造函数默认）且当前速度 < 目标速度 × 1.01）：
>     先按摩擦力对齐方向（`Velocity -= (Velocity - Dir × Speed) × Friction × Δt`），
>     再计算加速度 `(MoveVelocity - Velocity) / Δt`（上限 `MaxAccel`）——效果与玩家输入的转向感相同，不是瞬时赋值。
>   - **瞬时分支**（当前速度已达目标，或 `bRequestedMoveUseAcceleration = false`）：
>     `Velocity = MoveVelocity` 直接赋值，立即到达目标速度。
>   因此 `false` 模式并非"总是瞬时"——从静止加速时走加速分支（有加速感）；巡航速度维持方向时走瞬时分支（响应更直接）。
> - `true`：CrowdFollowingComponent 调用 `RequestPathMove(MoveInput)` →
>   `PawnOwner->Internal_AddMovementInput(MoveInput)` → `PendingInputVector`
>   → `CalcVelocity` 中的标准输入加速度管线消费。
>
> **共同结论**：两条路径都在 `CalcVelocity` 内被消费，`Super::CalcVelocity()` 返回后
> `Velocity` 已是确定值，排斥力在其之后叠加对两种模式均有效。
>
> 唯一体感差异：`true` 模式下基础寻路经标准输入管线整体更平滑；排斥力注入是帧级
> 瞬时的（`Velocity +=`），在两种模式下排斥响应都比基础寻路"更硬"——
> 这对防穿模场景是期望行为（即时响应优先于平滑过渡）。
> 宿主项目根据自身移动体感需求自行选择，插件无需适配。

```cpp
/**
 * 宿主项目的战斗单位 Actor 需要实现此接口。
 * 插件中所有需要读取单位属性的地方均通过此接口访问，
 * 不依赖任何具体的宿主类。
 *
 * 实现方式：C++ 继承 或 蓝图实现接口函数，两者均可。
```

### 6.2 TsnTacticalMovementComponent —— 合法排斥力注入

```cpp
// Public/Components/TsnTacticalMovementComponent.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/CharacterMovementComponent.h"
#include "TsnTacticalMovementComponent.generated.h"

class UTsnStanceRepulsionSubsystem;

/**
 * 自定义角色移动组件。
 *
 * 提供合法的排斥力注入接口，在 CalcVelocity（Super 完成后）统一叠加。
 * UE 5.7 中无论使用普通 PathFollowing 还是 DetourCrowd，CalcVelocity 均在
 * PhysWalking 中每帧调用，因此单一注入点即可覆盖所有路径。
 *
 * 站桩排斥力采用“至少 1 帧延迟消费”契约：
 * Subsystem 在帧 N 写入，MovementComponent 在帧 N+1 的首次 CalcVelocity 中消费，
 * 从而彻底消除 Tick 先后不确定性。
 *
 * 第三层防线——站桩穿透防护（Stance Penetration Guard）：
 * CalcVelocity 最终速度确定后，裁剪所有朝 ObstacleRadius 内 Stance 单位的向内分量。
 */
UCLASS()
class TIREFLYSQUADNAVIGATION_API UTsnTacticalMovementComponent
    : public UCharacterMovementComponent
{
    GENERATED_BODY()

public:
    UTsnTacticalMovementComponent();

    virtual void BeginPlay() override;
    virtual void EndPlay(const EEndPlayReason::Type EndPlayReason) override;

    /**
     * 由外部排斥力系统每帧调用，累加当前帧产生的排斥速度增量。
     * 采用累加语义（+=），允许多个系统在同一帧内安全写入。
     * 该值不会尝试同帧生效，而是在下一帧首次进入
     * CalcVelocity（Super 之后）时消费一次后清零。
     *
     * @param InRepulsionVelocity 排斥速度增量（调用方已乘 DeltaTime）
     */
    UFUNCTION(BlueprintCallable, Category = "TireflySquadNavigation|Avoidance")
    void SetRepulsionVelocity(const FVector& InRepulsionVelocity);

    /** 排斥力占最大速度的上限比例，防止排斥力完全压过寻路意图 */
    UPROPERTY(EditAnywhere, Category = "TireflySquadNavigation|Avoidance",
              meta = (ClampMin = "0.0", ClampMax = "1.0"))
    float MaxRepulsionVelocityRatio = 0.4f;

    /**
     * 排斥力叠加后允许的超速比例。
     * 1.0 = 不允许超速，1.1 = 允许 10% 超速（推荐，确保紧凑空间能被有效推开）。
     */
    UPROPERTY(EditAnywhere, Category = "TireflySquadNavigation|Avoidance",
              meta = (ClampMin = "1.0", ClampMax = "1.5"))
    float OverspeedToleranceRatio = 1.1f;

    /**
     * 是否绘制每帧消费的排斥力向量（箭头，橙色）。
        * 历史注记：这是 V2 方案中的组件级 raw debug 快照；当前插件已将运行时单位调试统一收敛到 UTsnDebugDrawSubsystem。
     * 在 Shipping 配置中此属性仍存在，但实际绘制代码不会编译进去。
     */
    UPROPERTY(EditAnywhere, Category = "TireflySquadNavigation|Debug")
    bool bDrawDebugRepulsion = false;

    /**
     * 对象池释放回调。宿主项目在将 Owner 返回对象池时调用此方法。
     * 等价于 EndPlay 中的清理逻辑，但不依赖 Actor 实际销毁。
     */
    UFUNCTION(BlueprintCallable, Category = "TireflySquadNavigation|ObjectPool")
    void OnOwnerReleased();

protected:
    /**
     * 排斥力唯一注入点。
     * UE 5.7 中无论 PathFollowing 还是 CrowdFollowing 最终都经过 PhysWalking → CalcVelocity，
     * 因此在 Super 完成后统一叠加排斥力，无需额外重写 RequestDirectMove。
     */
    virtual void CalcVelocity(
        float DeltaTime, float Friction,
        bool bFluid, float BrakingDeceleration) override;

private:
    /** 延迟到下一帧消费的排斥速度增量（消费后清零） */
    FVector DeferredRepulsionVelocity = FVector::ZeroVector;

    /** 是否有待处理的延迟排斥力 */
    bool bHasDeferredRepulsion = false;

    /** 记录排斥力首次写入发生的帧号，后续累加不更新此值（保证消费检测正确） */
    uint64 DeferredRepulsionWriteFrame = 0;

    /** 缓存 RepulsionSubsystem 指针，避免每帧 GetSubsystem 查询 */
    UPROPERTY()
    UTsnStanceRepulsionSubsystem* CachedRepulsionSubsystem = nullptr;

    /** 对排斥向量做限幅处理 */
    FVector GetClampedRepulsion() const;

    /** 当前帧是否允许消费延迟排斥力 */
    bool CanConsumeDeferredRepulsion() const;

    /** 消费延迟排斥力并清零 */
    void ConsumeDeferredRepulsion();

    /**
     * 站桩穿透防护（第三层防线）。
     * 查询 CachedRepulsionSubsystem 的 Stance 单位列表，对 ObstacleRadius 内的单位
     * 裁剪最终 Velocity 中朝向该单位的分量至零。
     */
    void ApplyStancePenetrationGuard();
};
```

```cpp
// Private/Components/TsnTacticalMovementComponent.cpp
#include "Components/TsnTacticalMovementComponent.h"
#include "Subsystems/TsnStanceRepulsionSubsystem.h"
#include "Components/TsnStanceObstacleComponent.h"
#include "TsnLog.h"

UTsnTacticalMovementComponent::UTsnTacticalMovementComponent()
{
    // ★ 从根源关闭 CMC 内置 RVO Avoidance，防止与 DetourCrowd 双重避障冲突
    bUseRVOAvoidance = false;
}

void UTsnTacticalMovementComponent::BeginPlay()
{
    Super::BeginPlay();

    // 向排斥力子系统注册为战术移动单位，使 Tick 遍历范围收窄为战术单位
    if (UTsnStanceRepulsionSubsystem* RepSys =
            GetWorld()->GetSubsystem<UTsnStanceRepulsionSubsystem>())
    {
        RepSys->RegisterMovingUnit(GetOwner());
        CachedRepulsionSubsystem = RepSys;
    }
}

void UTsnTacticalMovementComponent::EndPlay(const EEndPlayReason::Type EndPlayReason)
{
    if (CachedRepulsionSubsystem)
    {
        CachedRepulsionSubsystem->UnregisterMovingUnit(GetOwner());
    }

    CachedRepulsionSubsystem = nullptr;
    Super::EndPlay(EndPlayReason);
}

void UTsnTacticalMovementComponent::OnOwnerReleased()
{
    // 等价于 EndPlay 中的注销逻辑，供对象池回收时调用
    if (CachedRepulsionSubsystem)
    {
        CachedRepulsionSubsystem->UnregisterMovingUnit(GetOwner());
    }
    CachedRepulsionSubsystem = nullptr;
    DeferredRepulsionVelocity = FVector::ZeroVector;
    bHasDeferredRepulsion = false;
    DeferredRepulsionWriteFrame = 0;
}

void UTsnTacticalMovementComponent::SetRepulsionVelocity(const FVector& InRepulsionVelocity)
{
    // 累加语义：允许多个系统在同一帧内安全写入
    // 调试检测：仅在不同系统同帧调用时输出警告
    // （TsnStanceRepulsionSubsystem 内部已合并后只调用一次，不会触发此警告）
    ensureMsgf(
        !bHasDeferredRepulsion || DeferredRepulsionWriteFrame != GFrameCounter,
        TEXT("SetRepulsionVelocity called multiple times in frame %llu by different systems. "
             "Values will be accumulated. Check if multiple subsystems are injecting repulsion."),
        GFrameCounter);

    DeferredRepulsionVelocity += InRepulsionVelocity;

    // ★ 关键：仅在本周期首次写入时记录 WriteFrame，后续累加不更新。
    // UE 5.7 中 UTickableWorldSubsystem（FTickableGameObject）在所有 TickGroup
    // 执行完毕后才调度，因此 RepulsionSubsystem 天然晚于 CMC（TG_PrePhysics）；
    // 1 帧延迟由引擎调度顺序保证。首次写入保护是额外防御，
    // 防止未来 Tick 机制变更时产生同帧覆盖。
    if (!bHasDeferredRepulsion)
    {
        DeferredRepulsionWriteFrame = GFrameCounter;
    }
    bHasDeferredRepulsion = true;
}

FVector UTsnTacticalMovementComponent::GetClampedRepulsion() const
{
    float MaxRepulsionSpeed = MaxWalkSpeed * MaxRepulsionVelocityRatio;
    return DeferredRepulsionVelocity.GetClampedToMaxSize(MaxRepulsionSpeed);
}

bool UTsnTacticalMovementComponent::CanConsumeDeferredRepulsion() const
{
    return bHasDeferredRepulsion
        && DeferredRepulsionWriteFrame < GFrameCounter
        && !DeferredRepulsionVelocity.IsNearlyZero();
}

void UTsnTacticalMovementComponent::ConsumeDeferredRepulsion()
{
    DeferredRepulsionVelocity = FVector::ZeroVector;
    bHasDeferredRepulsion = false;
    DeferredRepulsionWriteFrame = 0;
}

// 唯一排斥力注入点：Super 完成后叠加 + 穿透防护
// UE 5.7 中 PhysWalking 每帧必调 CalcVelocity（CrowdFollowing 的 RequestedVelocity 也在此消费）
void UTsnTacticalMovementComponent::CalcVelocity(
    float DeltaTime, float Friction, bool bFluid, float BrakingDeceleration)
{
    // ① 父类正常计算（路径跟随/Crowd 的 RequestedVelocity 都在这里被处理）
    Super::CalcVelocity(DeltaTime, Friction, bFluid, BrakingDeceleration);

    // ② 父类计算完成后叠加“上一帧写入”的排斥力
    if (CanConsumeDeferredRepulsion())
    {
        Velocity += GetClampedRepulsion();

        // 限幅：允许轻微超速以确保排斥效果
        float SpeedCap = MaxWalkSpeed * OverspeedToleranceRatio;
        if (Velocity.Size2D() > SpeedCap)
        {
            Velocity = Velocity.GetSafeNormal2D() * SpeedCap
                     + FVector(0.f, 0.f, Velocity.Z);
        }

        ConsumeDeferredRepulsion();
    }

    // ③ 第三层防线：裁剪朝 Stance 单位 ObstacleRadius 内的向内速度分量
    ApplyStancePenetrationGuard();
}

void UTsnTacticalMovementComponent::ApplyStancePenetrationGuard()
{
    if (!CachedRepulsionSubsystem) return;

    const TArray<FTsnStanceObstacle>& StanceUnits = CachedRepulsionSubsystem->GetStanceUnits();
    if (StanceUnits.Num() == 0) return;

    FVector MyLoc = GetOwner()->GetActorLocation();
    FVector Vel2D = FVector(Velocity.X, Velocity.Y, 0.f);

    // 每帧通过 Obs.Unit->GetActorLocation() 动态读取位置（FTsnStanceObstacle 不缓存位置快照）。
    // 这样即使站桩单位因技能击退或 Root Motion 产生了位移，本帧拿到的仍是最新世界坐标。
    for (const FTsnStanceObstacle& Obs : StanceUnits)
    {
        if (!Obs.Unit.IsValid()) continue;

        FVector Diff = MyLoc - Obs.Unit->GetActorLocation();
        float Dist2D = Diff.Size2D();

        // 只对已穿入 ObstacleRadius（NavModifierRadius）的单位做裁剪
        if (Dist2D >= Obs.NavModifierRadius || Dist2D <= KINDA_SMALL_NUMBER) continue;

        FVector TowardStance2D = -Diff.GetSafeNormal2D();
        float InwardComponent = FVector::DotProduct(Vel2D, TowardStance2D);

        if (InwardComponent > 0.f)
        {
            Vel2D -= TowardStance2D * InwardComponent;

            // 第三层裁剪属于正常运行路径（排斥力柔性设计的必然补偿），用 Verbose 而非 Warning。
            // 开发阶段可通过过滤 LogTireflySquadNav Verbose 追踪前两层介入频率，辅助调参。
            UE_LOG(LogTireflySquadNav, Verbose,
                TEXT("StancePenetrationGuard: clipped inward velocity (%.1f cm/s) toward %s (dist2D=%.1f, ObstacleR=%.1f)"),
                InwardComponent, *GetNameSafe(Obs.Unit.Get()), Dist2D, Obs.NavModifierRadius);
        }
    }

    Velocity.X = Vel2D.X;
    Velocity.Y = Vel2D.Y;
}
```

```cpp
#include "CoreMinimal.h"
#include "Components/ActorComponent.h"
#include "TsnEngagementSlotComponent.generated.h"

/**
 * 槽位信息：挂在目标上的本地极坐标（角度 + 半径），不是持久缓存的世界坐标。
 */

{
    GENERATED_BODY()

    /** 占用者 */
    TWeakObjectPtr<AActor> Occupant;

    /** 在目标周围的本地方位角（度，0~360） */
    float AngleDeg = 0.f;

    /** 本地环半径（由占用者的 GetEngagementRange() 决定） */
    float Radius = 100.f;
};

/**
 * 交战槽位组件 —— 挂载在可被攻击的目标身上。
 *

 * 1. 多环支持：不同攻击距离的攻击者站在不同的环上
 * 2. 角度分配：从请求者当前位置计算理想角度，在该角度附近
 *    寻找不与同环已有攻击者冲突的空位

public:

     * 请求一个交战槽位。

     * @param Radius        环的半径

    float FindUnoccupiedAngle(float IdealAngleDeg, float Radius) const;

    /**
     * 计算指定本地角度和半径在目标当前位置下的世界空间快照。

     * @param AngleDeg 角度（度）
     * @param Radius   半径（cm）
     * @return 世界空间快照

    FVector CalculateWorldPosition(float AngleDeg, float Radius) const;

    /** 清理已失效的槽位引用（Occupant 被销毁等） */
    void CleanupInvalidSlots();
```

```cpp
// Private/Components/TsnEngagementSlotComponent.cpp
#include "Components/TsnEngagementSlotComponent.h"
#include "Subsystems/TsnEngagementSlotSubsystem.h"
#include "DrawDebugHelpers.h"
#include "TsnLog.h"

UTsnEngagementSlotComponent::UTsnEngagementSlotComponent()
{
    PrimaryComponentTick.bCanEverTick = false;
}

FVector UTsnEngagementSlotComponent::RequestSlot(AActor* Requester, float AttackRange)
{
    if (!Requester || !GetOwner())
        return GetOwner() ? GetOwner()->GetActorLocation() : FVector::ZeroVector;

    CleanupInvalidSlots();

    // ⓪ 通过子系统检查：若请求者已在其他目标上占有槽位，自动释放旧槽位
    if (UWorld* World = GetWorld())
    {
        if (auto* SlotSS = World->GetSubsystem<UTsnEngagementSlotSubsystem>())
        {
            UTsnEngagementSlotComponent* OldSlotComp = SlotSS->FindOccupiedSlotComponent(Requester);
            if (OldSlotComp && OldSlotComp != this)
            {
                OldSlotComp->ReleaseSlot(Requester);
            }
        }
    }

    // ① 已有槽位 → 检查攻击距离是否发生变化
    if (FTsnEngagementSlotInfo* Existing = FindExistingSlot(Requester))
    {
        float NewRadius = FMath::Max(AttackRange + SlotRadiusOffset, 10.f);
        if (FMath::Abs(Existing->Radius - NewRadius) > SameRingRadiusTolerance)
        {
            // 攻击距离变化超过容差 → 释放旧槽位，重新分配
            ReleaseSlot(Requester);
        }
        else
        {
            // 复用本地极坐标，重新计算当前世界快照
            return CalculateWorldPosition(Existing->AngleDeg, Existing->Radius);
        }
    }

    // ② 槽位已满 → 面向请求者方向的合理位置（不阻塞攻击）
    if (Slots.Num() >= MaxSlots)
    {
        float Radius = FMath::Max(AttackRange + SlotRadiusOffset, 10.f);
        FVector Dir = (Requester->GetActorLocation()
                       - GetOwner()->GetActorLocation()).GetSafeNormal2D();
        float FallbackAngle = FMath::RadiansToDegrees(FMath::Atan2(Dir.Y, Dir.X));
        return CalculateWorldPosition(FallbackAngle, Radius);
    }

    // ③ 计算环半径和理想角度
    float Radius = FMath::Max(AttackRange + SlotRadiusOffset, 10.f);
    FVector Dir = (Requester->GetActorLocation()
                   - GetOwner()->GetActorLocation()).GetSafeNormal2D();
    float IdealAngleDeg = FMath::RadiansToDegrees(FMath::Atan2(Dir.Y, Dir.X));

    // ④ 找不冲突的空位角度
    float BestAngle = FindUnoccupiedAngle(IdealAngleDeg, Radius);

    // ⑤ 注册槽位并向子系统登记
    FTsnEngagementSlotInfo NewSlot;
    NewSlot.Occupant = Requester;
    NewSlot.AngleDeg = BestAngle;
    NewSlot.Radius = Radius;
    Slots.Add(NewSlot);

    if (UWorld* World = GetWorld())
    {
        if (auto* SlotSS = World->GetSubsystem<UTsnEngagementSlotSubsystem>())
        {
            SlotSS->RegisterSlotOccupancy(Requester, this);
        }
    }

    return CalculateWorldPosition(BestAngle, Radius);
}

void UTsnEngagementSlotComponent::ReleaseSlot(AActor* Requester)
{
    int32 Removed = Slots.RemoveAll([Requester](const FTsnEngagementSlotInfo& Slot)
    {
        return Slot.Occupant == Requester;
    });

    // 若确实释放了槽位，通知子系统注销占用记录
    if (Removed > 0)
    {
        if (UWorld* World = GetWorld())
        {
            if (auto* SlotSS = World->GetSubsystem<UTsnEngagementSlotSubsystem>())
            {
                SlotSS->UnregisterSlotOccupancy(Requester);
            }
        }
    }
}

bool UTsnEngagementSlotComponent::HasSlot(AActor* Requester) const
{
    for (const FTsnEngagementSlotInfo& Slot : Slots)
    {
        if (Slot.Occupant == Requester) return true;
    }
    return false;
}

int32 UTsnEngagementSlotComponent::GetOccupiedSlotCount() const
{
    int32 Count = 0;
    for (const FTsnEngagementSlotInfo& Slot : Slots)
    {
        if (Slot.Occupant.IsValid()) ++Count;
    }
    return Count;
}

bool UTsnEngagementSlotComponent::IsSlotAvailable(AActor* Requester) const
{
    // 已持有槽位者始终可用（RequestSlot 会复用或重分配环位）
    if (HasSlot(Requester)) return true;
    return Slots.Num() < MaxSlots;
}

float UTsnEngagementSlotComponent::FindUnoccupiedAngle(
    float IdealAngleDeg, float Radius) const
{
    // 收集同环上已被占据的角度
    TArray<float> OccupiedAngles;
    for (const FTsnEngagementSlotInfo& Slot : Slots)
    {
        if (!Slot.Occupant.IsValid()) continue;
        if (FMath::Abs(Slot.Radius - Radius) < SameRingRadiusTolerance)
        {
            OccupiedAngles.Add(Slot.AngleDeg);
        }
    }

    if (OccupiedAngles.Num() == 0)
    {
        return IdealAngleDeg;
    }

    // 基于半径计算最小角度间隔（弧长 = 半径 × 弧度）
    float MinAngularGapDeg = FMath::RadiansToDegrees(MinSlotSpacing / Radius);
    MinAngularGapDeg = FMath::Max(MinAngularGapDeg, 25.f); // 下限保护

    // 从理想角度出发，正负交替搜索空位
    float SearchStep = MinAngularGapDeg * 0.5f;
    for (float Offset = 0.f; Offset <= 180.f; Offset += SearchStep)
    {
        float Candidates[2] = { IdealAngleDeg + Offset, IdealAngleDeg - Offset };
        int32 NumCandidates = (Offset == 0.f) ? 1 : 2;

        for (int32 i = 0; i < NumCandidates; ++i)
        {
            bool bConflict = false;
            for (float Occupied : OccupiedAngles)
            {
                if (FMath::Abs(FMath::FindDeltaAngleDegrees(Candidates[i], Occupied))
                    < MinAngularGapDeg)
                {
                    bConflict = true;
                    break;
                }
            }
            if (!bConflict) return Candidates[i];
        }
    }

    // 环被占满 → 返回理想角度（不阻塞攻击）
    return IdealAngleDeg;
}

FVector UTsnEngagementSlotComponent::CalculateWorldPosition(
    float AngleDeg, float Radius) const
{
    // AngleDeg 为世界空间方位角（0° = +X 轴），不随目标旋转
    float AngleRad = FMath::DegreesToRadians(AngleDeg);
    FVector OwnerLoc = GetOwner()->GetActorLocation();
    return OwnerLoc + FVector(FMath::Cos(AngleRad), FMath::Sin(AngleRad), 0.f) * Radius;
}

void UTsnEngagementSlotComponent::CleanupInvalidSlots()
{
    Slots.RemoveAll([](const FTsnEngagementSlotInfo& Slot) {
        return !Slot.Occupant.IsValid();
    });
}

void UTsnEngagementSlotComponent::OnOwnerReleased()
{
    // 双层 null-guard：
    // - GetWorld() 可能为 null（组件脱离 World 时）
    // - GetSubsystem() 可能为 null（切关卡时 Subsystem 先于组件被销毁）
    // 切关卡时对象池批量回收 + 关卡拆除同帧发生是最易触发此路径的场景，
    // 与 EndPlay 的 null-guard 原因相同，OnOwnerReleased 同样需要相同保护。
    if (UWorld* World = GetWorld())
    {
        if (auto* SlotSS = World->GetSubsystem<UTsnEngagementSlotSubsystem>())
        {
            for (const FTsnEngagementSlotInfo& Slot : Slots)
            {
                if (Slot.Occupant.IsValid())
                {
                    SlotSS->UnregisterSlotOccupancy(Slot.Occupant.Get());
                }
            }
        }
    }

    Slots.Empty();
}

FTsnEngagementSlotInfo* UTsnEngagementSlotComponent::FindExistingSlot(AActor* Requester)
{
    for (FTsnEngagementSlotInfo& Slot : Slots)
    {
        if (Slot.Occupant == Requester) return &Slot;
    }
    return nullptr;
}
```

```cpp
// Private/Subsystems/TsnEngagementSlotSubsystem.cpp
#include "Subsystems/TsnEngagementSlotSubsystem.h"
#include "Components/TsnEngagementSlotComponent.h"

void UTsnEngagementSlotSubsystem::RegisterSlotOccupancy(
    AActor* Requester, UTsnEngagementSlotComponent* SlotComp)
{
    if (!Requester || !SlotComp) return;
    SlotOccupancyMap.Add(Requester, SlotComp);
}

void UTsnEngagementSlotSubsystem::UnregisterSlotOccupancy(AActor* Requester)
{
    if (!Requester) return;
    SlotOccupancyMap.Remove(Requester);
}

UTsnEngagementSlotComponent* UTsnEngagementSlotSubsystem::FindOccupiedSlotComponent(
    AActor* Requester) const
{
    if (!Requester) return nullptr;
    if (const TWeakObjectPtr<UTsnEngagementSlotComponent>* Found = SlotOccupancyMap.Find(Requester))
    {
        return Found->Get();
    }
    return nullptr;
}
```

> **与 `TsnEngagementSlotComponent` 的协作**：
> `RequestSlot` 开头调用 `FindOccupiedSlotComponent(Requester)`——若返回非 null 且不是自身，
> 先调用旧组件的 `ReleaseSlot(Requester)`；分配成功后调用 `RegisterSlotOccupancy`。
> `ReleaseSlot` 在移除本地记录后调用 `UnregisterSlotOccupancy`。
> `OnOwnerReleased` 遍历所有 Slots，逐个通知子系统注销。

```cpp
void UTsnStanceObstacleComponent::GetSanitizedRadii(
    float& OutObstacleRadius, float& OutRepulsionRadius) const
{
    OutObstacleRadius = FMath::Max(ObstacleRadius, 10.f);

    const bool bValidOuterBand = RepulsionRadius > OutObstacleRadius;
    ensureMsgf(bValidOuterBand,
        TEXT("TsnStanceObstacleComponent: RepulsionRadius (%.1f) must be > ObstacleRadius (%.1f)."),
        RepulsionRadius, OutObstacleRadius);

    OutRepulsionRadius = bValidOuterBand
        ? RepulsionRadius
        : (OutObstacleRadius + 30.f);
}

void UTsnStanceObstacleComponent::InitNavModifier()
{
    float EffectiveObstacleRadius = 0.f;
    float IgnoredRepulsionRadius = 0.f;
    GetSanitizedRadii(EffectiveObstacleRadius, IgnoredRepulsionRadius);

    // ★ 开关式：BeginPlay 预创建，初始状态关闭，后续只做开关
    NavModifierComp = NewObject<UNavModifierComponent>(
        GetOwner(), UNavModifierComponent::StaticClass(),
        TEXT("TsnStanceObstacleNavMod"));

    NavModifierComp->RegisterComponent();
    NavModifierComp->FailsafeExtent =
        FVector(EffectiveObstacleRadius, EffectiveObstacleRadius, 100.f);
    ApplyNavAreaClass();
    NavModifierComp->SetCanEverAffectNavigation(false); // 初始关闭
}

void UTsnStanceObstacleComponent::ApplyNavAreaClass()
{
    if (!NavModifierComp) return;

    switch (NavModifierMode)
    {
        case ETsnNavModifierMode::Impassable:
            NavModifierComp->SetAreaClass(UNavArea_Null::StaticClass());
            break;
        case ETsnNavModifierMode::HighCost:
            NavModifierComp->SetAreaClass(UTsnNavArea_StanceUnit::StaticClass());
            break;
    }
}

void UTsnStanceObstacleComponent::ActivateNavModifier()
{
    if (!NavModifierComp) return;

    GetWorld()->GetTimerManager().ClearTimer(NavModifierDeactivationTimer);
    NavModifierComp->SetCanEverAffectNavigation(true);

    if (UNavigationSystemV1* NavSys =
            FNavigationSystem::GetCurrent<UNavigationSystemV1>(GetWorld()))
    {
        NavSys->UpdateComponentInNavOctree(*NavModifierComp);
    }
}

void UTsnStanceObstacleComponent::DeactivateNavModifier()
{
    if (!NavModifierComp) return;

    NavModifierComp->SetCanEverAffectNavigation(false);

    if (UNavigationSystemV1* NavSys =
            FNavigationSystem::GetCurrent<UNavigationSystemV1>(GetWorld()))
    {
        NavSys->UpdateComponentInNavOctree(*NavModifierComp);
    }
}

void UTsnStanceObstacleComponent::EnterStanceMode()
{
    if (CurrentMobilityStance == ETsnMobilityStance::Stance) return;
    CurrentMobilityStance = ETsnMobilityStance::Stance;

    // 补缓存（BeginPlay 时 Controller 可能还未 Possess）
    if (!CrowdFollowingComp) CacheComponents();

    if (CrowdFollowingComp)
        CrowdFollowingComp->SetCrowdSimulationState(ECrowdSimulationState::ObstacleOnly);

    if (bUseNavModifier)
        ActivateNavModifier();

    float EffectiveObstacleRadius = 0.f;
    float EffectiveRepulsionRadius = 0.f;
    GetSanitizedRadii(EffectiveObstacleRadius, EffectiveRepulsionRadius);

    if (UTsnStanceRepulsionSubsystem* RepSys =
            GetWorld()->GetSubsystem<UTsnStanceRepulsionSubsystem>())
    {
        RepSys->RegisterStanceUnit(
            GetOwner(), EffectiveRepulsionRadius, RepulsionStrength, EffectiveObstacleRadius);
    }
}

void UTsnStanceObstacleComponent::ExitStanceMode()
{
    if (CurrentMobilityStance == ETsnMobilityStance::Moving) return;
    CurrentMobilityStance = ETsnMobilityStance::Moving;

    if (CrowdFollowingComp)
        CrowdFollowingComp->SetCrowdSimulationState(ECrowdSimulationState::Enabled);

    // ★ 延迟关闭：避免其他单位路径突然跳变；对象池回收时（bIsReleasingOwner）改为立即关闭
    if (bUseNavModifier && NavModifierComp)
    {
        if (bIsReleasingOwner)
        {
            // 对象池回收路径：立即关闭，不产生定时器，防止定时器在下次重用时意外触发
            DeactivateNavModifier();
        }
        else
        {
            GetWorld()->GetTimerManager().SetTimer(
                NavModifierDeactivationTimer, this,
                &UTsnStanceObstacleComponent::DeactivateNavModifier,
                NavModifierDeactivationDelay, false);
        }
    }

    if (UTsnStanceRepulsionSubsystem* RepSys =
            GetWorld()->GetSubsystem<UTsnStanceRepulsionSubsystem>())
    {
        RepSys->UnregisterStanceUnit(GetOwner());
    }
}

void UTsnStanceObstacleComponent::UpdateStanceUnitParams()
{
    float EffectiveObstacleRadius = 0.f;
    float EffectiveRepulsionRadius = 0.f;
    GetSanitizedRadii(EffectiveObstacleRadius, EffectiveRepulsionRadius);

    // 更新 NavModifier 的障碍范围
    if (bUseNavModifier && NavModifierComp)
    {
        NavModifierComp->FailsafeExtent =
            FVector(EffectiveObstacleRadius, EffectiveObstacleRadius, 100.f);

        if (UNavigationSystemV1* NavSys =
                FNavigationSystem::GetCurrent<UNavigationSystemV1>(GetWorld()))
        {
            NavSys->UpdateComponentInNavOctree(*NavModifierComp);
        }
    }

    // 仅在 Stance 状态时同步 RepulsionSubsystem 缓存参数
    if (CurrentMobilityStance == ETsnMobilityStance::Stance)
    {
        if (UTsnStanceRepulsionSubsystem* RepSys =
                GetWorld()->GetSubsystem<UTsnStanceRepulsionSubsystem>())
        {
            RepSys->UpdateStanceUnit(
                GetOwner(), EffectiveRepulsionRadius, RepulsionStrength, EffectiveObstacleRadius);
        }
    }
}

void UTsnStanceObstacleComponent::OnOwnerReleased()
{
    bIsReleasingOwner = true;

    // 对象池回收：强制退出 Stance 状态。
    // ExitStanceMode 内部因 bIsReleasingOwner=true 走立即关闭路径，不产生定时器。
    if (CurrentMobilityStance == ETsnMobilityStance::Stance)
    {
        ExitStanceMode();
    }

    // 兜底：清理可能在本次回收前已存在的残留定时器（如上一次 ExitStanceMode 延迟尚未触发）
    GetWorld()->GetTimerManager().ClearTimer(NavModifierDeactivationTimer);

    bIsReleasingOwner = false;
}
```

```cpp
    AActor* Unit, float InRepulsionRadius, float InRepulsionStrength, float InNavModifierRadius)
{
    if (!Unit) return;
    for (const auto& Entry : StanceUnits)
    {
        // 幂等保护：已注册则跳过，防止 Blueprint 调用者连续两次 EnterStanceMode 产生重复条目。
        // EnterStanceMode 内部有 MobilityStance 幂等检查，正常路径不会触发此处；
        // 此保护仅针对绕过 BT/StateTree 的直接 Blueprint 调用。

    }
    StanceUnits.Add({ Unit, InRepulsionRadius, InRepulsionStrength, InNavModifierRadius });
}

    // 幂等：RemoveAll 对未注册的 Actor 无副作用；同时顺带清理失效弱引用。
    StanceUnits.RemoveAll([Unit](const FTsnStanceObstacle& Obs)
    {
        return Obs.Unit == Unit || !Obs.Unit.IsValid();
    });
}

    AActor* Unit, float InRepulsionRadius, float InRepulsionStrength, float InNavModifierRadius)
{
    if (!Unit) return;
    for (FTsnStanceObstacle& Entry : StanceUnits)
    {
        if (Entry.Unit == Unit)
        {
            Entry.RepulsionRadius = InRepulsionRadius;
            Entry.RepulsionStrength = InRepulsionStrength;
            Entry.NavModifierRadius = InNavModifierRadius;
            return;
        }
    }
    // 未注册的单位调用此方法无副作用
}

}

void UTsnStanceRepulsionSubsystem::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    CleanupInvalidEntries();
    if (StanceUnits.Num() == 0) return;

    UWorld* World = GetWorld();
    if (!World) return;

#if ENABLE_DRAW_DEBUG
    if (bDrawDebugRepulsion || GDrawDebugTsnRepulsion)
    {
        // 对每个 Stance 单位绘制双圆环：内层红色（NavModifierRadius）、外层黄色（RepulsionRadius）
        for (const FTsnStanceObstacle& Obs : StanceUnits)
        {
            if (!Obs.Unit.IsValid()) continue;
            FVector StanceLoc = Obs.Unit->GetActorLocation();
            DrawDebugCircle(World, StanceLoc, Obs.NavModifierRadius, 32,
                FColor::Red, false, -1.f, 0, 1.f,
                FVector::RightVector, FVector::ForwardVector, false);
            DrawDebugCircle(World, StanceLoc, Obs.RepulsionRadius, 32,
                FColor::Yellow, false, -1.f, 0, 1.f,
                FVector::RightVector, FVector::ForwardVector, false);
        }
    }
#endif

    // 清理失效的移动单位
    MovingUnits.RemoveAll([](const TWeakObjectPtr<AActor>& Ptr) { return !Ptr.IsValid(); });

    for (const TWeakObjectPtr<AActor>& WeakUnit : MovingUnits)
    {
        AActor* MovingActor = WeakUnit.Get();
        if (!MovingActor) continue;

        ACharacter* MovingChar = Cast<ACharacter>(MovingActor);
        if (!MovingChar) continue;

        // 只处理移动中的战斗单位（实现了接口且当前非 Stance 状态）
        UTsnStanceObstacleComponent* ObsComp =
            MovingChar->FindComponentByClass<UTsnStanceObstacleComponent>();
        if (!ObsComp || ObsComp->GetMobilityStance() != ETsnMobilityStance::Moving)
            continue;

        UTsnTacticalMovementComponent* MoveComp =
            Cast<UTsnTacticalMovementComponent>(MovingChar->GetCharacterMovement());
        if (!MoveComp) continue;

        FVector MyLoc = MovingChar->GetActorLocation();
        FVector TotalRepulsion = FVector::ZeroVector;

        for (const FTsnStanceObstacle& Obs : StanceUnits)
        {
            if (!Obs.Unit.IsValid() || Obs.Unit.Get() == MovingChar) continue;

            FVector Diff = MyLoc - Obs.Unit->GetActorLocation();
            float Dist2D = Diff.Size2D();

            if (Dist2D >= Obs.RepulsionRadius || Dist2D <= KINDA_SMALL_NUMBER) continue;

            // ==========================================
            // 双阶段力度计算（核心）
            // ==========================================
            float ForceMagnitude;

            if (Dist2D < Obs.NavModifierRadius)
            {
                // 内层：已穿入 NavModifier 区域（异常情况）→ 全力排斥
                float Penetration = 1.f - (Dist2D / Obs.NavModifierRadius);
                ForceMagnitude = Obs.RepulsionStrength * FMath::Square(Penetration);
            }
            else
            {
                // 外层：NavModifier 已覆盖路径规划 → 极微弱线性引导
                float OuterRange = Obs.RepulsionRadius - Obs.NavModifierRadius;
                float OuterDist = Dist2D - Obs.NavModifierRadius;
                float OuterFactor = 1.f - (OuterDist / OuterRange);
                ForceMagnitude = Obs.RepulsionStrength * OuterRepulsionRatio * OuterFactor;
            }

            // 方向：TangentBlendRatio 切线引导 + (1 - TangentBlendRatio) 径向排斥
            FVector RadialDir = Diff.GetSafeNormal2D();
            FVector CurrentMoveDir = MoveComp->Velocity.GetSafeNormal2D();

            FVector ForceDir;
            if (!CurrentMoveDir.IsNearlyZero())
            {
                FVector Tangent = FVector::CrossProduct(FVector::UpVector, CurrentMoveDir);
                float Side = FVector::DotProduct(RadialDir, Tangent);
                FVector TangentDir = Tangent * FMath::Sign(Side);
                ForceDir = (TangentDir * TangentBlendRatio
                          + RadialDir * (1.f - TangentBlendRatio)).GetSafeNormal();
            }
            else
            {
                ForceDir = RadialDir;
            }

            TotalRepulsion += ForceDir * ForceMagnitude;
        }

        if (!TotalRepulsion.IsNearlyZero())
        {
            // ★ 通过合法接口注入，不直接修改 Velocity；
            //   当前帧只写入延迟缓冲，下一帧首次移动更新时统一消费
            MoveComp->SetRepulsionVelocity(TotalRepulsion * DeltaTime);

#if ENABLE_DRAW_DEBUG
            if (bDrawDebugRepulsion || GDrawDebugTsnRepulsion)
            {
                // 对每个受排斥的移动单位绘制排斥力方向箭头（橙色）
                DrawDebugDirectionalArrow(
                    World,
                    MovingActor->GetActorLocation(),
                    MovingActor->GetActorLocation() + TotalRepulsion.GetSafeNormal() * 80.f,
                    20.f, FColor::Orange, false, -1.f, 0, 2.f);
            }
#endif
        }
    }
}
```

```cpp
void UTsnUnitSeparationComponent::TickComponent(
    float DeltaTime, ELevelTick TickType, FActorComponentTickFunction* ThisTickFunction)
{
    Super::TickComponent(DeltaTime, TickType, ThisTickFunction);

    APawn* MyPawn = Cast<APawn>(GetOwner());
    if (!MyPawn || !CachedRepulsionSubsystem) return;

    // 只对移动中的单位生效
    UTsnStanceObstacleComponent* ObsComp =
        MyPawn->FindComponentByClass<UTsnStanceObstacleComponent>();
    if (ObsComp && ObsComp->GetMobilityStance() != ETsnMobilityStance::Moving) return;

    FVector MyLoc = MyPawn->GetActorLocation();
    FVector SeparationDir = FVector::ZeroVector;
    TArray<FVector> InwardToStanceDirs;

    // Moving↔Moving：遍历子系统注册列表做直接距离查询，避免物理 Overlap 开销
    // MovingUnits 包含所有已注册的战术单位（含 Stance），需手动过滤当前为 Stance 的项
    for (const TWeakObjectPtr<AActor>& WeakOther : CachedRepulsionSubsystem->GetMovingUnits())
    {
        AActor* Other = WeakOther.Get();
        if (!Other || Other == MyPawn) continue;

        // 跳过当前处于 Stance 的单位（Stance↔Moving 由 RepulsionSubsystem 独占负责）
        UTsnStanceObstacleComponent* OtherObsComp =
            Other->FindComponentByClass<UTsnStanceObstacleComponent>();
        if (OtherObsComp && OtherObsComp->GetMobilityStance() == ETsnMobilityStance::Stance)
            continue;

        FVector Diff = MyLoc - Other->GetActorLocation();
        float Dist2D = Diff.Size2D();
        if (Dist2D <= KINDA_SMALL_NUMBER || Dist2D >= SeparationRadius) continue;

        float Factor = 1.f - (Dist2D / SeparationRadius);
        SeparationDir += Diff.GetSafeNormal2D() * Factor;
    }

    // Stance 约束方向：来自 StanceUnits 列表，仅记录方向供后续裁剪
    for (const FTsnStanceObstacle& Obs : CachedRepulsionSubsystem->GetStanceUnits())
    {
        if (!Obs.Unit.IsValid()) continue;
        FVector StanceLoc = Obs.Unit->GetActorLocation();
        float Dist2D = (MyLoc - StanceLoc).Size2D();
        if (Dist2D >= SeparationRadius || Dist2D <= KINDA_SMALL_NUMBER) continue;
        // 朝内方向 = 从自身指向站桩单位（进入内侧的方向）
        InwardToStanceDirs.Add((StanceLoc - MyLoc).GetSafeNormal2D());
    }

    FVector AdjustedSeparation = SeparationDir.GetClampedToMaxSize(1.f);
    if (InwardToStanceDirs.Num() > 0)
    {
        // 累加所有站桩单位的 InwardToStance 方向，归一化后做**单次**投影裁剪
        // 这样不会因多个站桩单位从不同方向约束而过度裁剪 Separation
        FVector AccumulatedInward = FVector::ZeroVector;
        for (const FVector& InwardToStance : InwardToStanceDirs)
        {
            AccumulatedInward += InwardToStance;
        }
        AccumulatedInward = AccumulatedInward.GetSafeNormal2D();
        if (!AccumulatedInward.IsNearlyZero())
        {
            float InwardDot = FVector::DotProduct(AdjustedSeparation, AccumulatedInward);
            if (InwardDot > 0.f)
            {
                // 移除指向站桩单位内侧的分量，只允许切向/外向的软协调输入保留
                AdjustedSeparation -= AccumulatedInward * InwardDot;
            }
        }
    }

    if (!AdjustedSeparation.IsNearlyZero())
    {
        // 通过 AddMovementInput 注入；进入站桩 keep-out 邻域时，
        // 该输入已先被裁剪，不会把单位继续推向站桩单位。
        //
        // 【DeltaTime 不参与此注入】
        // AddMovementInput 仅将 SeparationStrength 写入 PendingMovementInputVector，
        // 不乘以本帧 DeltaTime。最终速度积分 (Velocity += Accel * DeltaSeconds) 由 CMC
        // 在其自己的 TickComponent 中完成，与本组件的 30Hz tick 间隔无关。
        // 因此 30Hz → 60Hz 帧率变化不会导致分离力脉冲放大或缩小。
        MyPawn->AddMovementInput(AdjustedSeparation.GetSafeNormal2D(), SeparationStrength);

#if ENABLE_DRAW_DEBUG
        if (bDrawDebugSeparation)
        {
            FVector Base = MyPawn->GetActorLocation();
            // 裁剪后的最终分离方向（绿色）
            DrawDebugDirectionalArrow(
                GetWorld(), Base,
                Base + AdjustedSeparation.GetSafeNormal2D() * 80.f,
                20.f, FColor::Green, false, -1.f, 0, 2.f);
            // 裁剪前的原始分离方向（如有裁剪，显示为蓝色对比）
            FVector RawDir = SeparationDir.GetSafeNormal2D();
            if (!RawDir.IsNearlyZero() && !RawDir.Equals(AdjustedSeparation.GetSafeNormal2D(), 0.1f))
            {
                DrawDebugDirectionalArrow(
                    GetWorld(), Base,
                    Base + RawDir * 80.f,
                    20.f, FColor::Blue, false, -1.f, 0, 1.f);
            }
        }
#endif
    }
}

void UTsnUnitSeparationComponent::OnOwnerReleased()
{
    SetComponentTickEnabled(false);
    CachedRepulsionSubsystem = nullptr;
}
```

#### 6.7.1 未来扩展：何时引入全局力合并器

当前方案**不引入**通用“全局力合并器”，原因是现阶段只有两类语义清晰的输入：

- `TsnStanceRepulsionSubsystem`：Moving↔Stance 的**硬约束 / keep-out**
- `TsnUnitSeparationComponent`：Moving↔Moving 的**软协调 / spacing**

通过“职责拆分 + 站桩约束下裁剪 Separation 朝内分量”即可得到确定行为，复杂度最低。

但如果后续新增了更多**同帧需要仲裁**的 steering 源，例如：

- 编队保持 / 队形校正
- 技能击退 / 吸附 / 脚本位移偏置
- 临时战术偏移（例如侧移、掩体贴边、让路）
- 需要统一调试可视化、权重统计或可解释仲裁的多个力源

此时再扩展为全局力合并器会更合适，因为仅靠“局部裁剪 + 系数竞争”将难以稳定描述最终行为。

**扩展方式建议**：

1. 在 `TsnTacticalMovementComponent` 内引入统一的 steering 收集结构（如 `FTsnSteeringContribution`）。
2. 定义明确语义层级（例如 `HardKeepOut` → `NavigationAssist` → `SoftCoordination`）。
3. 各系统不再直接 `AddMovementInput()` / `SetRepulsionVelocity()`，而是先提交贡献项。
4. MovementComponent 在每帧固定顺序解析：先处理硬约束，再叠加导航辅助，最后叠加软协调输入。
5. 保留当前接口作为适配层，逐步迁移，避免一次性重写所有调用方。

这意味着：**只有当力源数量与仲裁复杂度真实增长时，例如出现更多同层 steering 源（编队保持、脚本推挤、技能击退、局部战术偏移），并且都要求同一帧可解释仲裁，才把当前“轻量分层规则”升级为“全局合并框架”。**

```cpp
EBTNodeResult::Type UTsnBTTask_ChaseEngagementTarget::ExecuteTask(
    UBehaviorTreeComponent& OwnerComp, uint8* NodeMemory)
{
    AAIController* AICon = OwnerComp.GetAIOwner();
    if (!AICon || !AICon->GetPawn()) return EBTNodeResult::Failed;

    AActor* Target = GetTargetActor(OwnerComp);
    if (!Target) return EBTNodeResult::Failed;

    // 从接口读取攻击距离——未实现接口或返回非法值时直接 Failed
    APawn* Pawn = AICon->GetPawn();
    if (!Pawn->Implements<UTsnTacticalUnit>())
    {
        UE_LOG(LogTireflySquadNav, Warning,
            TEXT("TsnBTTask_ChaseEngagementTarget: Pawn [%s] does not implement ITsnTacticalUnit. Returning Failed."),
            *Pawn->GetName());
        return EBTNodeResult::Failed;
    }

    CachedAttackRange = ITsnTacticalUnit::Execute_GetEngagementRange(Pawn);
    if (CachedAttackRange <= 0.f)
    {
        UE_LOG(LogTireflySquadNav, Warning,
            TEXT("TsnBTTask_ChaseEngagementTarget: GetEngagementRange() returned %.1f (must be > 0). Returning Failed."),
            CachedAttackRange);
        return EBTNodeResult::Failed;
    }
    CachedPreEngagementRadius = CachedAttackRange * PreEngagementRadiusMultiplier;

    // 已在预战斗距离内（含已在攻击距离内）→ 无需追击
    if (FVector::Dist2D(Pawn->GetActorLocation(), Target->GetActorLocation())
        <= CachedPreEngagementRadius)
    {
        return EBTNodeResult::Succeeded;
    }

    FAIMoveRequest MoveReq;
    MoveReq.SetGoalActor(Target);
    MoveReq.SetAcceptanceRadius(CachedPreEngagementRadius);
    MoveReq.SetUsePathfinding(true);

    FPathFollowingRequestResult Result = AICon->MoveTo(MoveReq);

    if (Result.Code == EPathFollowingRequestResult::AlreadyAtGoal)
        return EBTNodeResult::Succeeded;
    if (Result.Code == EPathFollowingRequestResult::Failed)
        return EBTNodeResult::Failed;

    return EBTNodeResult::InProgress;
}

void UTsnBTTask_ChaseEngagementTarget::TickTask(
    UBehaviorTreeComponent& OwnerComp, uint8* NodeMemory, float DeltaSeconds)
{
    AAIController* AICon = OwnerComp.GetAIOwner();
    if (!AICon || !AICon->GetPawn())
    {
        FinishLatentTask(OwnerComp, EBTNodeResult::Failed);
        return;
    }

    AActor* Target = GetTargetActor(OwnerComp);
    if (!Target)
    {
        AICon->StopMovement();
        FinishLatentTask(OwnerComp, EBTNodeResult::Failed);
        return;
    }

    float Dist = FVector::Dist2D(AICon->GetPawn()->GetActorLocation(),
                                  Target->GetActorLocation());
    if (Dist <= CachedPreEngagementRadius)
    {
        AICon->StopMovement();
        FinishLatentTask(OwnerComp, EBTNodeResult::Succeeded);
    }
    // MoveToActor 自动跟踪目标位置变化，无需手动重寻路
}

EBTNodeResult::Type UTsnBTTask_ChaseEngagementTarget::AbortTask(
    UBehaviorTreeComponent& OwnerComp, uint8* NodeMemory)
{
    if (AAIController* AICon = OwnerComp.GetAIOwner()) AICon->StopMovement();
    return EBTNodeResult::Aborted;
}
```

```cpp
bool UTsnBTTask_MoveToEngagementSlot::RequestSlotAndMove(
    UBehaviorTreeComponent& OwnerComp, AActor* Target)
{
    AAIController* AICon = OwnerComp.GetAIOwner();
    if (!AICon || !AICon->GetPawn()) return false;

    UTsnEngagementSlotComponent* SlotComp =
        Target->FindComponentByClass<UTsnEngagementSlotComponent>();
    if (!SlotComp) return false;

    // 槽满检查：返回 false 让 BT 树进入失败分支、重新选择目标
    if (!SlotComp->IsSlotAvailable(AICon->GetPawn()))
    {
        UE_LOG(LogTireflySquadNav, Log,
            TEXT("MoveToEngagementSlot: Target [%s] slots are full, returning Failed."),
            *Target->GetName());
        return false;
    }

    CachedSlotPosition = SlotComp->RequestSlot(AICon->GetPawn(), CachedAttackRange);

    FAIMoveRequest MoveReq;
    MoveReq.SetGoalLocation(CachedSlotPosition);
    MoveReq.SetAcceptanceRadius(AcceptanceRadius);
    MoveReq.SetUsePathfinding(true);

    FPathFollowingRequestResult Result = AICon->MoveTo(MoveReq);
    CachedTargetLocation = Target->GetActorLocation();
    TimeSinceLastRePathCheck = 0.f;

    // MoveTo 失败时释放已分配的槽位，防止泄漏
    if (Result.Code == EPathFollowingRequestResult::Failed)
    {
        SlotComp->ReleaseSlot(AICon->GetPawn());
        return false;
    }
    return true;
}

EBTNodeResult::Type UTsnBTTask_MoveToEngagementSlot::ExecuteTask(
    UBehaviorTreeComponent& OwnerComp, uint8* NodeMemory)
{
    AAIController* AICon = OwnerComp.GetAIOwner();
    AActor* Target = GetTargetActor(OwnerComp);
    if (!AICon || !AICon->GetPawn() || !Target) return EBTNodeResult::Failed;

    APawn* Pawn = AICon->GetPawn();

    // 接口合法性检查：未实现接口或返回非法值时直接 Failed
    if (!Pawn->Implements<UTsnTacticalUnit>())
    {
        UE_LOG(LogTireflySquadNav, Warning,
            TEXT("TsnBTTask_MoveToEngagementSlot: Pawn [%s] does not implement ITsnTacticalUnit. Returning Failed."),
            *Pawn->GetName());
        return EBTNodeResult::Failed;
    }

    CachedAttackRange = ITsnTacticalUnit::Execute_GetEngagementRange(Pawn);
    if (CachedAttackRange <= 0.f)
    {
        UE_LOG(LogTireflySquadNav, Warning,
            TEXT("TsnBTTask_MoveToEngagementSlot: GetEngagementRange() returned %.1f (must be > 0). Returning Failed."),
            CachedAttackRange);
        return EBTNodeResult::Failed;
    }

    // 已在攻击距离内 → 直接进入站桩攻击，不分配槽位
    if (FVector::Dist2D(Pawn->GetActorLocation(), Target->GetActorLocation())
        <= CachedAttackRange)
        return EBTNodeResult::Succeeded;

    ElapsedApproachTime = 0.f;
    bIsFinishing = false;    // 每次任务启动时重置，防止上次执行的残留状态干扰
    CachedOwnerComp = &OwnerComp;
    CachedTarget = Target;   // 缓存目标，供 OnTaskFinished 解绑时使用

    // 绑定目标失效委托：OnDestroyed（真正销毁）+ OnEndPlay（对象池回收等场景）
    Target->OnDestroyed.AddDynamic(this, &UTsnBTTask_MoveToEngagementSlot::OnTargetDestroyed);
    Target->OnEndPlay.AddDynamic(this, &UTsnBTTask_MoveToEngagementSlot::OnTargetEndPlay);

    return RequestSlotAndMove(OwnerComp, Target)
        ? EBTNodeResult::InProgress : EBTNodeResult::Failed;
}

void UTsnBTTask_MoveToEngagementSlot::TickTask(
    UBehaviorTreeComponent& OwnerComp, uint8* NodeMemory, float DeltaSeconds)
{
    AAIController* AICon = OwnerComp.GetAIOwner();
    if (!AICon || !AICon->GetPawn())
    {
        FinishLatentTask(OwnerComp, EBTNodeResult::Failed);
        return;
    }

    AActor* Target = GetTargetActor(OwnerComp);
    if (!Target)
    {
        ReleaseCurrentSlot(OwnerComp);
        AICon->StopMovement();
        FinishLatentTask(OwnerComp, EBTNodeResult::Failed);
        return;
    }

    FVector PawnLoc = AICon->GetPawn()->GetActorLocation();

    // 退出条件 (a)：到达槽位
    if (FVector::Dist2D(PawnLoc, CachedSlotPosition) <= AcceptanceRadius)
    {
        AICon->StopMovement();
        FinishLatentTask(OwnerComp, EBTNodeResult::Succeeded);
        return;
    }

    // 退出条件 (b)：中途进入攻击距离（目标向自身靠近）
    if (FVector::Dist2D(PawnLoc, Target->GetActorLocation()) <= CachedAttackRange)
    {
        AICon->StopMovement();
        FinishLatentTask(OwnerComp, EBTNodeResult::Succeeded);
        return;
    }

    // 退出条件 (c)：超时安全兜底（路径异常兜底）
    ElapsedApproachTime += DeltaSeconds;
    if (MaxApproachTime > 0.f && ElapsedApproachTime >= MaxApproachTime)
    {
        AICon->StopMovement();
        FinishLatentTask(OwnerComp, EBTNodeResult::Failed);
        return;
    }

    // 周期性检查：若目标已明显移动，则基于已持有槽位的本地参数刷新世界快照并重发 MoveTo
    TimeSinceLastRePathCheck += DeltaSeconds;
    if (TimeSinceLastRePathCheck >= RePathCheckInterval)
    {
        TimeSinceLastRePathCheck = 0.f;
        if (FVector::Dist2D(CachedTargetLocation, Target->GetActorLocation())
            > RePathDistanceThreshold)
        {
            RequestSlotAndMove(OwnerComp, Target);
        }
    }
}

EBTNodeResult::Type UTsnBTTask_MoveToEngagementSlot::AbortTask(
    UBehaviorTreeComponent& OwnerComp, uint8* NodeMemory)
{
    ReleaseCurrentSlot(OwnerComp);
    if (AAIController* AICon = OwnerComp.GetAIOwner()) AICon->StopMovement();
    return EBTNodeResult::Aborted;
}

void UTsnBTTask_MoveToEngagementSlot::ReleaseCurrentSlot(
    UBehaviorTreeComponent& OwnerComp)
{
    AActor* Target = GetTargetActor(OwnerComp);
    AAIController* AICon = OwnerComp.GetAIOwner();
    if (!Target || !AICon || !AICon->GetPawn()) return;

    if (UTsnEngagementSlotComponent* SlotComp =
            Target->FindComponentByClass<UTsnEngagementSlotComponent>())
    {
        SlotComp->ReleaseSlot(AICon->GetPawn());
    }
}

void UTsnBTTask_MoveToEngagementSlot::OnTaskFinished(
    UBehaviorTreeComponent& OwnerComp, uint8* NodeMemory,
    EBTNodeResult::Type TaskResult)
{
    // 使用缓存的目标指针解绑委托，而非从 Blackboard 读取——
    // 目标被销毁时 BB 可能已清除引用，GetTargetActor 会返回 nullptr 导致解绑被跳过。
    if (AActor* Target = CachedTarget.Get())
    {
        Target->OnDestroyed.RemoveDynamic(
            this, &UTsnBTTask_MoveToEngagementSlot::OnTargetDestroyed);
        Target->OnEndPlay.RemoveDynamic(
            this, &UTsnBTTask_MoveToEngagementSlot::OnTargetEndPlay);
    }
    CachedTarget.Reset();
    CachedOwnerComp.Reset();
    Super::OnTaskFinished(OwnerComp, NodeMemory, TaskResult);
}

void UTsnBTTask_MoveToEngagementSlot::OnTargetDestroyed(AActor* DestroyedActor)
{
    // Actor 销毁时 OnEndPlay 必先于 OnDestroyed 触发，bIsFinishing 已被置 true，直接 return
    if (bIsFinishing) return;
    bIsFinishing = true;

    UBehaviorTreeComponent* OwnerComp = CachedOwnerComp.Get();
    if (!OwnerComp) return;

    ReleaseCurrentSlot(*OwnerComp);
    if (AAIController* AICon = OwnerComp->GetAIOwner())
    {
        AICon->StopMovement();
    }
    FinishLatentTask(*OwnerComp, EBTNodeResult::Failed);
}

void UTsnBTTask_MoveToEngagementSlot::OnTargetEndPlay(
    AActor* Actor, EEndPlayReason::Type EndPlayReason)
{
    // 防双触发：OnEndPlay 可能触发两次（对象池回收 + 真正销毁），或与 OnDestroyed 先后触发
    if (bIsFinishing) return;
    bIsFinishing = true;

    UBehaviorTreeComponent* OwnerComp = CachedOwnerComp.Get();
    if (!OwnerComp) return;

    ReleaseCurrentSlot(*OwnerComp);
    if (AAIController* AICon = OwnerComp->GetAIOwner())
    {
        AICon->StopMovement();
    }
    FinishLatentTask(*OwnerComp, EBTNodeResult::Failed);
}
```

## 七·补、StateTree 决策层

插件提供与 BT 节点等价的 StateTree 决策能力，方便宿主项目直接在 StateTree 资产里组装 TSN 战斗循环，而无需通过 BT 包装。

### 7·补.1 节点清单

| 类型 | 节点 | 说明 |
|------|------|------|
| Task | `FTsnStateTreeTask_ChaseEngagementTarget` | 追击目标到预战斗距离 |
| Task | `FTsnStateTreeTask_MoveToEngagementSlot` | 申请并接近交战槽位（成功保留槽位、失败/中断释放） |
| Task | `FTsnStateTreeTask_EnterStanceMode` | 切换为站姿模式 |
| Task | `FTsnStateTreeTask_ExitStanceMode` | 切换为移动模式 |
| Task | `FTsnStateTreeTask_ReleaseEngagementSlot` | 显式释放槽位（目标缺失也返回 Succeeded） |
| Evaluator | `FTsnStateTreeEvaluator_CombatContext` | 输出 `bHasValidTarget`、`DistanceToTarget2D`、`EngagementRange`、`PreEngagementRadius`、`bIsInEngagementRange`、`bIsInPreEngagementRange` |
| Evaluator | `FTsnStateTreeEvaluator_TargetMotion` | 输出 `TargetSpeed2D`、`bIsTargetStationary` |
| Condition | `FTsnStateTreeCondition_HasValidTarget` | 消费 `bHasValidTarget` |
| Condition | `FTsnStateTreeCondition_IsInPreEngagementRange` | 消费 `DistanceToTarget2D + PreEngagementRadius` |
| Condition | `FTsnStateTreeCondition_IsInEngagementRange` | 消费 `DistanceToTarget2D + EngagementRange` |
| Condition | `FTsnStateTreeCondition_IsTargetStationary` | 消费 `bIsTargetStationary` |
| Condition | `FTsnStateTreeCondition_IsInStanceMode` | 直接查询 `UTsnStanceObstacleComponent::GetMobilityStance()` |

### 7·补.2 最小推荐状态链

```
Root (Evaluators: CombatContext, TargetMotion)
├── Idle [HasValidTarget == false]
└── Combat [HasValidTarget == true]
	├── Chase [!IsInPreEngagementRange]               → Task: ChaseEngagementTarget
	├── MoveToSlot [IsInPreEngagementRange && !IsInEngagementRange] → Task: MoveToEngagementSlot
	└── Engage [IsInEngagementRange]                  → Tasks: EnterStanceMode + 宿主攻击逻辑
		└── (退出时) Tasks: ReleaseEngagementSlot + ExitStanceMode
```

### 7·补.3 接入说明

- 推荐在 StateTree 资产中使用引擎自带的 `UStateTreeAIComponentSchema`（来自 `GameplayStateTreeModule`），它会自动提供 `AAIController` Context 字段；TSN 节点的 `AIController` 输入由 Schema 注入。
- `TargetActor` 字段需要由宿主项目通过 StateTree 资产参数或上层评估器绑定。
- `MoveToEngagementSlot` 在 ExitState 时会按“成功 vs 失败/中断”决定槽位是否保留；显式 `ReleaseEngagementSlot` 用于状态切换离开 Engage 时清理。
- 现有 BT 节点保持不变；BT 自动化测试不受 StateTree 节点影响。

### 7·补.4 测试模块中的 demo-only StateTree 节点

`TireflySquadNavigationTest` 模块额外提供两个仅用于演示 / 测试的 StateTree 任务：

| 类型 | 节点 | 说明 |
|------|------|------|
| Task | `FTsnTestStateTreeTask_SelectTarget` | 与测试模块中的 BT `SelectTarget` 等价，负责在测试地图里搜索敌方目标并写回共享参数 |
| Task | `FTsnTestStateTreeTask_Attack` | 纯测试用攻击计时器，达到 `AttackDuration` 后返回 `Succeeded` |

- 这两个节点只存在于 `EditorNoCommandlet` 的测试模块中，用于 Siege / Skirmish / MovingTarget 三个演示场景的最小 StateTree 复现。
- 宿主项目做正式接入时，应自行实现目标选择与攻击逻辑，而不是依赖测试模块节点。

## 八、每帧执行时序

> **UE 5.7 引擎事实补充**：
> - `bUseAccelerationForPaths` 默认为 `false`（`FNavMovementProperties` 初始化），
>   决定 CrowdFollowingComponent 以哪条路径向 CMC 投递速度：
>   - `false`：调用 `RequestDirectMove(NewVelocity)` → 写入 `RequestedVelocity + bHasRequestedVelocity`
>     → 下一帧 `CalcVelocity` → `ApplyRequestedMove` 消费。
>     （CMC 默认 `bRequestedMoveUseAcceleration = true`，所以 `ApplyRequestedMove` 在速度未达目标时仍做加速度过渡。）
>   - `true`：调用 `RequestPathMove(MoveInput)` → `PawnOwner->Internal_AddMovementInput`
>     → `PendingInputVector` → `CalcVelocity` 中的标准 Acceleration 管线消费。
> - `CalcVelocity()` **每帧都跑**——无论使用哪条投递路径。
>
> 结论：`RequestDirectMove` / `RequestPathMove` 和 `CalcVelocity` **不是互斥路径**；
> 前两者只是"输入端"，`CalcVelocity` 才是"消费端"。因此排斥力只需在
> `CalcVelocity`（`Super` 之后）**统一注入一次**，对两种模式均有效。

```
┌─ 两帧的执行流程（至少 1 帧延迟） ───────────────────────────────────────┐
│                                                                          │
│  帧 N：生产排斥力                                                        │
│  ├─ AIController / PathFollowing / UnitSeparation                        │
│  │   └─ 正常写入 PendingInputVector，或                                  │
│  │      CrowdFollowingComponent::RequestDirectMove() 写入               │
│  │      RequestedVelocity（将在帧 N+1 CalcVelocity 中被消费）            │
│  └─ TsnStanceRepulsionSubsystem::Tick()                                  │
│      └─ TsnTacticalMovementComponent::SetRepulsionVelocity(Force*Δt)    │
│          └─ 仅写入 DeferredRepulsionVelocity；                           │
│             仅在首次写入（bHasDeferredRepulsion==false）时记录             │
│             WriteFrame = N                                                │
│                                                                          │
│  帧 N+1：消费上一帧排斥力（统一在 CalcVelocity 中）                      │
│  ├─ Super::CalcVelocity()：                                              │
│  │   ├─ ApplyControlInputVector → 消费 PendingInputVector               │
│  │   └─ ApplyRequestedMove → 消费 RequestedVelocity（含 CrowdVelocity）│
│  └─ ★ Super 之后：                                                       │
│      ├─ 若 WriteFrame < CurrentFrame，叠加 DeferredRepulsionVelocity    │
│      │   限幅后清零                                                      │
│      └─ ApplyStancePenetrationGuard()：站桩穿透防护（正常触发路径）        │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
```

| 输入来源 | 注入方式 | 消费者 |
|---------|---------|--------|
| 路径跟随（PathFollowing） | `AddMovementInput()` → PendingInputVector | `Super::CalcVelocity()` 内部 `ApplyControlInputVector` |
| DetourCrowd 群体避障 | `RequestDirectMove()` → RequestedVelocity | `Super::CalcVelocity()` 内部 `ApplyRequestedMove` |
| 分离力（UnitSeparation） | `AddMovementInput()` → PendingInputVector | `Super::CalcVelocity()` 内部 `ApplyControlInputVector`（仅 Moving↔Moving；若邻近 Stance，先裁剪朝内分量） |
| 站桩排斥力（StanceRepulsion） | `SetRepulsionVelocity()` → DeferredRepulsionVelocity（首次写入时记录帧号） | 下一帧 `CalcVelocity()` 的 Super 之后（Moving↔Stance 唯一权威） |
| 站桩穿透防护 | `ApplyStancePenetrationGuard()` | `CalcVelocity()` 排斥力叠加之后（正常触发路径，为排斥力柔性设计的必然补偿） |

**各系统在注入点上互不冲突，但语义上并非完全平级：站桩排斥力是 keep-out 硬约束，分离力只是软协调输入。**

其中，站桩排斥力明确采用**至少 1 帧延迟消费**：Subsystem 只负责在帧 N 生产并写入数据，
MovementComponent 只在帧 N+1 的首次移动更新中消费。UE 5.7 中 `UTickableWorldSubsystem`
（`FTickableGameObject`）在所有 TickGroup（包括 CMC 所在的 `TG_PrePhysics`）执行完毕后
才被调度，因此 **RepulsionSubsystem 天然在 CMC 之后 Tick，1 帧延迟由引擎调度顺序保证**。
`WriteFrame` 首次写入保护是额外防御性措施，防止未来 Tick 机制变更时产生同帧覆盖。

同时，`UnitSeparation` 必须在产力阶段就遵守该硬约束：若附近存在站桩单位，任何“继续推向站桩单位内侧”的
分离分量都要被移除，只允许切向或远离站桩单位的输入继续参与移动融合。

## 九、NavArea 策略选择

| 战场类型 | 推荐策略 | NavModifierMode 配置 |
|---------|---------|---------------------|
| 棋盘格 / 规整开阔地（自走棋） | **不可通行** | `Impassable`（默认） |
| 半开放地形（RTS / MOBA） | 高代价 | `HighCost`，`DefaultCost = 50` |
| 复杂室内 / 走廊 | 高代价 + 加强排斥力 | `HighCost`，提高 `RepulsionStrength` |

**使用 Impassable 的前提**：地图保证站桩单位的 NavModifier 区域不会将目标完全封死（即始终有绕行路径）。建议：
- `MaxSlots` 限制同时交战人数
- `ObstacleRadius` = 角色碰撞半径 × 1.2~1.5，不要过大

> **2026-05 LegendsTD 集成注记**
>
> 上面的倍率化经验值适合通用 demo 和早期原型，不适合 LegendsTD 当前这种“近战固定 1 格、双方可能互为目标”的对称接敌模型。对这类宿主，后续实现应改用“编辑器可见的默认预算 + 显式硬约束”，而不是继续依赖 `1.35` 一类隐藏倍率。
>
> 推荐的配置链路是：
> 1. 宿主 `Project Settings` 决定胶囊半径、格子边长和近战攻击距离。
> 2. TSN 插件 `UTsnDeveloperSettings` 提供默认 obstacle / nav modifier / repulsion 几何预算。
> 3. 单位上的 `UTsnStanceObstacleComponent` 在 `bOverrideTsnDefaults` 打开时按需 override 默认值。
>
> 对 1 格近战，建议把下面这些关系视为硬语义约束：
> - `CapsuleRadius <= ObstacleRadius < NavModifierRadius <= RepulsionRadius < EngagementRange`
> - `ObstacleRadius - CapsuleRadius <= 10cm`
> - `RepulsionRadius - ObstacleRadius <= 10cm`
> - `NavModifierRadius - ObstacleRadius` 只保留很薄的一层外扩，建议 `0~5cm`
> - `EngagementRange - RepulsionRadius >= CapsuleRadius`
> - 若双方会以同攻击距离互为目标，还应满足 `2 * RepulsionRadius <= EngagementRange`
>
> 如果这条最后约束不成立，那么两个近战单位的外层排斥环会天然相交，后续卡位问题通常不是调 steering 系数就能真正修掉的。

## 十、调参指南

| 组件 | 参数 | 含义 | 推荐值 | 调参方向 |
|------|------|------|--------|---------|
| `TsnTacticalMovementComponent` | `MaxRepulsionVelocityRatio` | 排斥力速度上限占最大移速的比例，防止排斥力完全压过寻路意图 | `0.3`~`0.5` | 配合 NavModifier 使用时调低；纯排斥力方案时调高 |
| | `OverspeedToleranceRatio` | 排斥力叠加后允许的超速比例（1.0 = 不允许，1.1 = 允许 10% 超速） | `1.1` | 密集场景可提高到 1.2，确保紧凑空间也能被推开 |
| `TsnStanceObstacleComponent` | `ObstacleRadius` | NavModifier 标记区域的半径，同时是排斥力双阶段内外层的分界线 | 碰撞半径 × 1.3 | 太大 → NavMesh 出现多余不可通行区；太小 → 绕行效果不明显 |
| | `RepulsionRadius` | 排斥力外层感知半径（外层弱引导的作用范围上界） | ObstacleRadius × `1.5`~`2.5` | **必须 > ObstacleRadius**；建议至少保留 `30cm` 外层带，非法配置运行时会自动修正 |
| | `RepulsionStrength` | 排斥力基础强度（cm/s 量级），内层满强度，外层上限 10% | `800` | 通常为 MaxWalkSpeed × 1.5~2.0 |
| | `NavModifierDeactivationDelay` | 退出站姿后延迟关闭 NavModifier 的时间，避免其他单位路径跳变 | `0.3s` | 太短 → 路径突变；太长 → NavModifier 残留干扰寻路 |
| | `bUseNavModifier` | 是否启用 NavModifier 影响路径规划（true = 双层防线，false = 仅排斥力） | `true` | 切换状态非常频繁（< 0.5s）时设 false，避免频繁 NavMesh Tile 重建 |
| `TsnEngagementSlotComponent` | `MaxSlots` | 最大同时交战单位数，超出时退化为面向目标的合理位置 | `8`~`12` | 近战为主设 8；远程为主可设更大 |
| | `SlotRadiusOffset` | 槽位半径相对攻击距离的偏移量（负值 = 比攻击距离更近） | `-20` | 确保站在槽位时严格在攻击范围内；绝对值过大可能造成穿模 |
| | `MinSlotSpacing` | 同一环上相邻槽位之间的最小间距（cm） | `80` | 根据角色碰撞胶囊直径调整，通常约等于碰撞直径 |
| | `SameRingRadiusTolerance` | 半径差在此范围内的攻击者被视为同一环参与角度冲突检测 | `60` | 建议等于碰撞胶囊直径，防止不同攻击距离单位被误判为同环 |
| `TsnBTTask_ChaseEngagementTarget` | `PreEngagementRadiusMultiplier` | 预战斗距离相对攻击距离的倍率，追击阶段在此距离停止并转入槽位接近 | `1.5` | 值越大战场展开越提前；过大会导致单位在较远处就停止追击、开始占位 |
| `TsnBTTask_MoveToEngagementSlot` | `RePathCheckInterval` | 周期性检查目标是否移动并刷新槽位世界快照的时间间隔（秒） | `0.1s` | 太短 → CPU 开销增加；太长 → 追踪移动目标时槽位位置漂移明显 |
| | `RePathDistanceThreshold` | 目标移动超过此距离时触发重新寻路（cm） | `50` | 建议 ≤ `2 × AcceptanceRadius`；以 `600cm/s` 目标为例，此设置可把单次典型漂移控制在约 `60cm` |
| | `MaxApproachTime` | 预占位接近阶段的最大允许时间，超时后强制进入后续步骤（秒，0 = 不限时） | `5.0s` | 太短可能导致单位未到位就停止；太长则路径异常时阻塞过久 |
| `TsnUnitSeparationComponent` | `SeparationRadius` | 分离力的感知半径，超出此范围的单位不产生分离力 | `120` | 约碰撞半径 2~3 倍；过大会导致单位无法密集站位 |
| | `SeparationStrength` | 分离力强度（作为 AddMovementInput 的 ScaleValue） | `0.5` | 0.3 → 柔和推开；0.8 → 明显推开；过大会干扰路径跟随方向 |
| `TsnStanceRepulsionSubsystem` | `TangentBlendRatio` | 排斥力的切向混合比例。值越高，排斥方向越偏向切线方向（绕行感更强）；值越低越偏向纯径向推开 | `0.7` | 0.5 → 平衡；0.7 → 推荐（明显绕行）；1.0 → 纯切向 |
| | `OuterRepulsionRatio` | 外层排斥力占内层满强度的比例上限。外层为渐变引导带，此参数控制外层推力的天花板 | `0.1` | 0.05 → 极微弱引导；0.1 → 推荐；0.3 → 较强引导，可能干扰寻路 |
| `TsnNavArea_StanceUnit` | `DefaultCost` | 穿越站桩单位区域的导航代价倍率（HighCost 模式下生效） | `10.0` | 5 → 轻微绕行；10 → 明显绕行；50 → 接近阻断 |

**碰撞通道设置**：

```cpp
// AI 战斗单位的碰撞胶囊：Pawn vs Pawn 设为 Overlap

// 对静态障碍物（墙壁等）仍然保持 Block
```

## 十三、性能优化

### 13.1 NavModifier 批量切换

多个单位在同一帧同时切换状态会触发多次 NavMesh Tile 重建，可随机延迟错开：

```cpp
void UTsnStanceObstacleComponent::EnterStanceModeDeferred()
{
    float RandomDelay = FMath::RandRange(0.f, 0.15f);
    GetWorld()->GetTimerManager().SetTimer(
        DeferredSwitchTimer, this,
        &UTsnStanceObstacleComponent::EnterStanceMode,
        RandomDelay, false);
}
```

### 13.2 排斥力系统分组 Tick

单位数量 > 50 时可将移动单位分组，每帧只处理一组：

```cpp
int32 CurrentGroup = GFrameCounter % NumGroups;
for (TActorIterator<ACharacter> It(World); It; ++It)
{
    if ((*It)->GetUniqueID() % NumGroups != CurrentGroup) continue;
    // ... 处理该单位
}
```

## 十四、扩展场景

### 14.1 目标被完全包围的处理

- `TsnNavArea_StanceUnit::DefaultCost = 10.0`（非 FLT_MAX），确保死路时仍可穿过
- `MaxSlots` 设置合理上限，满员后新单位转为等待或攻击别的目标
- 在行为树中增加槽位满检测：如无空闲槽位则等待或选其他目标

### 14.2 单位死亡时的自动清理

所有注册/注销逻辑均基于组件生命周期（`BeginPlay` / `EndPlay`），新 Spawn 的单位自动参与，被销毁的单位自动清理；`TWeakObjectPtr` 保证异步销毁时不产生悬空指针。

### 14.3 远程单位

- `GetEngagementRange()` 返回较大值（如 600），自动分配到外环
- `ObstacleRadius` 可相对较小（远程单位间距要求低）
- 行为上与近战单位完全一致，无需特殊处理
